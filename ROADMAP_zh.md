# 路线图

## 近期

- 增加更多攻击数据包，例如 RAG 污染、工具调用伪造、canary 重放
- 用结构化解析替代纯关键词规则，提升响应检测质量
- 增加跨提交的报告对比能力，方便回归分析
- 提供 Docker 和 compose，一条命令启动本地实验室

## 中期

- 为更多客户端预置 provider 适配器
- 在 UI 中增加请求/响应 diff 与命中原因解释
- 支持导出 markdown 和 SARIF 风格报告
- 增加多上游路由策略和环境配置

## 长期

- 形成探针与检测器插件系统
- 支持分布式 benchmark 执行
- 支持发现问题后的重放与语料最小化
